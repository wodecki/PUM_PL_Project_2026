# Konfiguracja środowiska

## Opcja 1: Google Colab (zalecana)

Google Colab to podstawowe obsługiwane środowisko. Nie wymaga lokalnej instalacji.

### Kroki

1. **Prześlij pliki danych** na Google Drive:
   ```
   My Drive/
   └── PUM/
       └── 2. data/
           ├── customers.csv
           └── transactions.csv
   ```

2. **Otwórz notatnik startowy** w Colab:
   - Prześlij `3. notatniki/mpN_starter.ipynb` do Colab, LUB
   - Prześlij na Google Drive i otwórz stamtąd

3. **Uruchom pierwszą komórkę** (blokady reprodukowalności):
   ```python
   import sklearn, pandas as pd, numpy as np
   # Asercje wersji przejdą na standardowym Colab
   ```
   Jeśli asercje nie przechodzą, zobacz sekcję Rozwiązywanie problemów poniżej.

4. **Uruchom drugą komórkę** (ładowanie danych):
   - Notatnik automatycznie wykrywa Colab i dostosowuje ścieżkę danych
   - Sprawdzenie sumy MD5 weryfikuje poprawność plików danych

### Ścieżka danych w Colab

Notatniki automatycznie wykrywają Colab i ustawiają:

Upewnij się, że Twoje pliki znajdują się dokładnie w tej ścieżce, lub zmodyfikuj ścieżkę w notatniku.

Jeśli zostaniesz poproszony, zamontuj dysk:

## Opcja 2: Konfiguracja lokalna

### Wymagania wstępne

- Python 3.10 lub nowszy
- Menedżer pakietów `uv` ([instrukcja instalacji](https://docs.astral.sh/uv/getting-started/installation/))

### Kroki

1. **Sklonuj lub pobierz** repozytorium

2. **Utwórz środowisko wirtualne i zainstaluj zależności**:
   ```bash
   cd PUM_PL
   uv venv
   uv pip install pandas numpy scikit-learn matplotlib seaborn jupyter
   ```

3. **Zweryfikuj wersje bibliotek**:
   ```bash
   uv run python -c "
   import sklearn, pandas, numpy
   print(f'sklearn:  {sklearn.__version__}')
   print(f'pandas:   {pandas.__version__}')
   print(f'numpy:    {numpy.__version__}')
   "
   ```
   Oczekiwane: sklearn 1.4.x–1.5.x, pandas 2.x, numpy <2.0

4. **Uruchom Jupyter**:
   ```bash
   uv run jupyter notebook "3. notatniki/"
   ```

5. **Otwórz notatnik startowy** i uruchom dwie pierwsze komórki, aby zweryfikować konfigurację

### Lokalna ścieżka danych

Podczas uruchamiania lokalnie notatniki ustawiają:
```python
DATA_DIR = Path("../2. data")
```

Działa to, jeśli uruchomisz Jupyter z katalogu `3. notatniki/`.

## Rozwiązywanie problemów

### „AssertionError: scikit-learn version..."

Twoja wersja scikit-learn jest poza wspieranym zakresem.

**Naprawa w Colab:**
```python
!pip install scikit-learn==1.5.2
# Następnie zrestartuj runtime: Runtime → Restart runtime
```

**Naprawa lokalna:**
```bash
uv pip install scikit-learn==1.5.2
```

### „AssertionError: numpy version..."

Masz zainstalowany numpy 2.x. Niektóre funkcje sklearn zachowują się inaczej z numpy 2.

**Naprawa:**
```python
!pip install numpy==1.26.4
# Zrestartuj runtime
```

### „AssertionError: customers.csv MD5 mismatch"

Twój plik danych jest uszkodzony lub pochodzi z innej wersji.

**Naprawa:** Pobierz ponownie `customers.csv` z materiałów kursowych. Poprawny hash MD5 to `c016cd4bfc36ac8d0a99bb6a3d55fa44`.

### „FileNotFoundError: data directory not found"

Notatnik nie może znaleźć Twoich plików danych.

**Colab:** Upewnij się, że pliki znajdują się w `My Drive/PUM/2. data/customers.csv`

**Lokalnie:** Upewnij się, że uruchamiasz Jupyter z katalogu `3. notatniki/`, aby ścieżka `../2. data/` rozwiązywała się poprawnie.

### „ModuleNotFoundError: No module named 'seaborn'"

**Colab:**
```python
!pip install seaborn
```

**Lokalnie:**
```bash
uv pip install seaborn
```

### Moje wyniki nie zgadzają się z oczekiwanymi odpowiedziami

1. Sprawdź, czy używasz `random_state=42` wszędzie
2. Sprawdź, czy postępujesz zgodnie z dokładną 12-krokową kolejnością przetwarzania w MP2
3. Uruchom `3. notatniki/skrypty/verify_checkpoint.py`, aby porównać wynik MP2 z bazowym punktem kontrolnym
4. Jeśli Twoje wyniki są bliskie, ale nie dokładne, sprawdź:
   - Czy posortowałeś kolumny alfabetycznie po kodowaniu?
   - Czy użyłeś `drop_first=False` w `pd.get_dummies()`?
   - Czy skonwertowałeś kolumny boolean z dummies na int?
   - Czy usunąłeś wartości odstające zarówno z X, jak i z y (oraz gender_series)?
